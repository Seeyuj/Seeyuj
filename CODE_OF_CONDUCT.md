## Objectif

Ce projet vise à fournir une **infrastructure open-source de simulation de mondes sandbox persistants**, maintenable sur le long terme.

Ce code de conduite a pour objectif de garantir :
- des échanges techniques constructifs ;
- un environnement de collaboration respectueux et professionnel ;
- la protection de la vision, de l’architecture et des mainteneurs du projet.

La participation au projet implique l’acceptation de ces règles.

---

## Principes généraux

### Respect et professionnalisme

Toute interaction (issues, pull requests, discussions, revues de code, commentaires) doit rester :

- respectueuse ;
- factuelle ;
- orientée solution ;
- centrée sur le contenu technique et architectural.

Les attaques personnelles, le mépris, l’ironie agressive ou les jugements sur les intentions ou compétences d’autrui ne sont pas tolérés.

---

### Désaccords techniques

Les désaccords sont normaux et attendus dans un projet d’infrastructure.

Ils doivent :
- s’appuyer sur des arguments techniques, mesurables ou architecturaux ;
- rester impersonnels ;
- respecter les principes fondamentaux du projet.

Un désaccord technique n’est jamais une attaque personnelle.

---

### Communication responsable

Les contributeurs doivent comprendre que :

- les mainteneurs contribuent sur leur temps ;
- il n’existe aucune obligation de réponse immédiate ;
- aucune contribution n’est garantie d’être acceptée ;
- aucune roadmap publique n’est contractuelle.

Toute tentative de pression, d’insistance répétée ou de contournement des décisions de gouvernance est contraire à l’esprit du projet.

---

## Comportements inacceptables

Les comportements suivants ne sont pas tolérés :

- attaques personnelles ou insinuations ;
- harcèlement, intimidation ou provocation ;
- débats idéologiques ou militants sans lien avec le projet ;
- remise en cause publique et répétée des décisions de gouvernance ;
- tentatives de forcer une orientation technique par volume ou insistance ;
- pollution des discussions par du hors-sujet ou des propositions manifestement incompatibles avec la vision du projet.

---

## Gouvernance et autorité du projet

Ce projet est open-source, mais **pas sans gouvernance**.

Les mainteneurs :
- sont garants de la vision et de l’architecture ;
- valident ou refusent les contributions ;
- arbitrent les désaccords techniques.

Une contribution peut être refusée :
- même si elle est techniquement correcte ;
- même si elle est populaire ;
- sans obligation de proposer une alternative ;
- sans débat prolongé.

Les décisions sont prises dans l’intérêt du projet à long terme, pas pour satisfaire un consensus immédiat.

---

## Modération et application

Les mainteneurs se réservent le droit de :

- modérer ou fermer une discussion ;
- fermer une issue ou une pull request ;
- demander des modifications de ton ou de contenu ;
- limiter ou suspendre l’accès aux contributions ;

lorsqu’un comportement nuit à la qualité du projet ou à son bon fonctionnement.

Ces mesures peuvent être appliquées sans avertissement préalable en cas de comportement manifestement problématique.

---

## Portée

Ce code de conduite s’applique à l’ensemble des espaces du projet :
- dépôt GitHub ;
- issues et pull requests ;
- discussions ;
- revues de code.

En participant au projet, vous acceptez ces règles.
